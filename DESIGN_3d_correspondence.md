# 立体対照ビュー 設計書（P7-5 / コンセプト1.1-3）

開発コンセプト「③立体模型との対照：作図はあくまで平面の構造式としつつ、『作図上ではこう書くが、実際の分子はこうなっている』という対照提示ができるとなおよい」を実現するための設計。実装前の合意用ドキュメント。

## 1. 基本方針

- **平面作図の正しさと立体的な実像を混同させない**ことが最重要。パズルの作図・判定には一切影響を与えず、**別枠の「対照表示」**として提供する。
- 外部ライブラリ（three.js等）は導入しない（開発方針2章）。化学の教科書で使われる**くさび形表記（wedge-dash）**を第1段階とし、必要ならSVG/CSSだけで実現できる疑似3Dへ段階的に進む。
- 表示は分子データから**自動生成**し、追加のコンテンツ作成を不要にする（プリセット立体配座のみ例外）。

## 2. 段階案

### M1（推奨・最初に実装）: sp3炭素の局所くさび図

- **何を見せるか**: 選択したsp3炭素1個を中心に、4つの置換基を教科書の慣習で描いた図：
  - 紙面内の結合2本（実線・上下方向）
  - 手前に出る結合1本（黒塗りくさび ▶）
  - 奥に向かう結合1本（破線くさび）
- **置換基の表現**: 単原子ならその元素記号、枝なら簡易式（-CH₃、-C₂H₅、-OH…）を `describeStructure`/断片コードから生成。
- **UI**: 右パネルに「🧊 立体で見る」ボタン → モーダル。左に現在の2D作図（対象炭素をハイライト）、右にくさび図。対象炭素はキャンバスでクリックして選ぶ（sp3炭素のみ選択可）。
- **教育文言**: 「作図では90°に描いていますが、実際のsp3炭素の結合角は約109.5°で、4つの置換基は正四面体の頂点方向に伸びています」。
- **不斉学習との連携**: 不斉炭素を選んだ場合、「4つの置換基がすべて異なる＝鏡像と重ならない」ことを明記。将来的には鏡像体のくさび図を並べて表示（R/S表記の導入口）。

### M2: 分子全体の標準くさび描画

- 主鎖をジグザグ（120°）で描き、各sp3炭素の残り2結合を上下のくさび/破線で自動配置した「教科書式の全体図」を対照表示。
- 正準コード（P8-2）から決定的にレイアウトすれば、ユーザーの描き方に依存しない標準図になる。
- 環はM2では六員環（いす形は描かず平面多角形＋くさび）まで。

### M3（将来）: 疑似3D回転表示

- CSS 3D transform または SVG投影の自作で、棒球モデルを回転させて見せる（依存なしで実現可能）。
- シクロヘキサンの**いす形配座**など、平面作図と実像の差が大きい代表例のプリセット（conformers.json）を用意し、パズル側の該当ステージから参照できるようにする。

## 3. 実装配置

- `stereo.js`（新設）: StereoView クラス。モーダル・くさび図SVG生成。chemistry.js の断片コード/describeStructure を流用。
- chemistry.js への追加は「sp3判定（既存の不斉判定の前提部分の切り出し）」程度に留め、純粋ロジックを維持。
- 検証・スナップ・座標系には一切手を入れない。

## 4. リスク・留意点

- くさび図の**見やすさ**（置換基ラベルの重なり）: M1は局所表示（中心1炭素）に限定して回避。
- **環内炭素**: M1では対象外（選択不可）とし、M2以降で扱う。
- R/S命名は本設計のスコープ外（将来、鏡像図と合わせて検討）。

## 5. マイルストーン（ロードマップ登録用）

- **P7-5-M3** 疑似3D回転とプリセット配座（いす形など）
