# イオンでみる化学反応式（Ion-Equation）開発方針

中学理科〜高校化学基礎向けの化学反応式学習アプリ。
反応式の係数や生成物を「丸暗記」ではなく、**水中のイオンの組み変わりを見て納得する**ことを目的とする。
vanilla JS + SVG、ビルドツールなし・静的配信（GitHub Pages 互換）。Chem-Assembler の姉妹プロジェクト。

## コンセプト（M1: 中和モード）

学習ループ:

1. **投入** — 酸・塩基の分子をボタンでビーカーに落とす
2. **電離** — 水面に達すると分子がイオンに分かれるアニメ（HCl → H⁺ + Cl⁻）。強電解質は完全電離
3. **反応** — 「反応させる」で H⁺ と OH⁻ がペアで引き合い、結合して H₂O が生まれる
4. **試行** — H⁺ が余れば「まだ酸性」。塩基を追加してちょうど中和を探す
5. **係数へ橋渡し** — 「H₂SO₄ 1個に NaOH 2個でちょうど中和」という体験がそのまま係数のヒントになる
6. **反応式完成** — 係数ステッパーを操作すると左右の原子数が即時集計され、つり合うと正解。
   同時にイオン反応式（H⁺ + OH⁻ → H₂O）と傍観イオンの解説を表示し、「塩は残ったイオンの組」であることを示す

クリア条件 = 「ビーカーでちょうど中和を達成」かつ「係数が最簡整数比でつり合う」の両方。

## 設計原則

- **model.js は DOM 非依存の純粋ロジック**（種の定義・電離表・原子数集計・係数判定）。UI から分離しテスト対象とする
- **粒子の座標・動きは見た目専用**。正否判定はイオンの個数と原子数のみで行う
  （Chem-Assembler の「検証はトポロジーのみ」と同じ思想）
- 正否判定は模範解答との文字列比較ではなく**原子数・電荷の保存＋最簡整数比**で行う（別解を機械的に許容できる形）
- データは v1 では JS 埋め込み（file:// 直開きでも動く）。JSON fetch を導入したらローカルサーバー必須になるので、その時点でこの記述を更新すること
- 全ファイル **UTF-8（BOMなし）**。コミット前に文字化けパターン（`縺`・`繧`・`繝`・`蜊`・`荳`・`邨`）がないか確認
- バージョン番号 `vNN` はコミットごとに **index.html・test.html のキャッシュバスター＋ヘッダー表示**を同時更新
- **1修正=1コミット**。修正ごとにブラウザで実挙動を検証し、検証内容をコミットメッセージに記録
- **コミット前に test.html を開いて全テスト合格を確認**

## ロードマップ

反応タイプを3群に大別し、群ごとにアニメーションエンジンを作る（2026-07-22 合意）。
アプリの軸は「反応式」。無機化学・理論化学への対応づけはステージの単元タグで行う（アプリを教科の枠に内包させない）。

- **A群: イオンの組み変わり**（電子は移動しない）— 中和・沈殿・気体発生。現行エンジンの拡張で賄う
- **B群: 電子の移動** — 酸化還元（金属+酸、イオン化傾向、電池・電気分解）。e⁻ 粒の新エンジン
- **C群: 分子の組み換え** — 化合・分解・燃焼。イオンが出ないため別ステージ形式（将来構想）

- [x] M1: 中和モード初版 — 強酸×強塩基3ステージ、電離・中和アニメ、係数チェック、イオン反応式表示（v1）
- [x] M1.5: 数合わせビュー — 反応式の直下に係数ぶんの粒を並べ、組み変えアニメで余り・不足を見せる（v2）
- [x] M1.6: 磨き込み — 粒子ドラッグ操作（イオンを相手に重ねて1組だけ反応）・モバイルのタップ標的・遊び方4ステップの折りたたみパネル（v13〜v15）
- [x] M2: 沈殿反応（A群）— ルール表 `stage.rules` 導入、沈殿の沈降・堆積アニメ、ステージ4（AgNO₃+NaCl）・5（BaCl₂+Na₂SO₄）（v5）
- [x] M3: 気体発生（A群）— ルールを多重集合対応（2H⁺+CO₃²⁻）に一般化、中間体 H₂CO₃ を一瞬表示→分解、CO₂ の泡が水面で逃げるアニメ、ステージ6（v6）
