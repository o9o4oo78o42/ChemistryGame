# アニメーション付きチュートリアル 設計書（P9-6）

入力デバイス別（マウス／タッチパッド／タッチパネル）の操作チュートリアル。
実キャンバス上でポインタの動きをデモ再生し、字幕で解説する。
呼び出しは邪魔にならない導線（❓ボタンのFAQ一覧＋検索、ボタンの長押しホバー）に限定する。
実装前の合意用ドキュメントであり、実装の進行に合わせて更新する。

---

## 1. 体験イメージ

1. ヘッダーの「❓ヘルプ」を押すと FAQ パネルが開く。パートごとの一覧
   （原子を置く／結合を描く／次数を変える・切る／伸ばす／環と縮合／官能基／反応を実行…）
   と検索ボックスが並ぶ
2. 項目の「▶ デモを見る」を押すとパネルが閉じ、**実際のキャンバス上で**
   ゴーストカーソル（光る円）が動いて操作を再現する。下部に字幕
   （例:「空いているマスをクリックすると炭素が置けます」）
3. デモは実イベント（PointerEvent）で本物のアプリを駆動するので、
   画面に起きることは本番の操作結果そのもの。終了後は元の作図状態に復元する
4. 操作ボタン（モジュール・ツール等）を **800ms 以上ホバー**すると
   「▶ この機能のデモを見る」の小さなチップが出る（クリックで該当デモへ）。
   通常のツールチップと同様、すぐ離せば何も出ない

## 2. デバイス別バリアント

- 再生する **イベント列は3デバイスで共通**（アプリはPointer Eventsに統一済み）。
  違いは **字幕の文言** と **カーソルの見た目** のみ:
  | デバイス | 判定 | カーソル表示 | 文言例 |
  |---|---|---|---|
  | マウス | `pointer: fine` かつ既定 | 矢印風の光点 | 「クリック」「右クリックで切断」 |
  | タッチパッド | 手動選択 | 矢印風の光点 | 「タップ」「2本指タップで切断」 |
  | タッチパネル | `pointer: coarse` | 指先風の大きめ光点 | 「タップ」「長押しで切断」「2本指でピンチ」 |
- 初期値はメディア特性で自動判定し、FAQパネル内のセレクタで手動切替できる
- 字幕データは `{ caption: { mouse, trackpad, touch } }` 形式。共通文はプレーン文字列で省略可

## 3. データモデル（tutorials.json 新設）

```json
[
  {
    "id": "place-atom",
    "title": "原子を置く・つなげる",
    "keywords": ["配置", "クリック", "炭素", "自動結合"],
    "steps": [
      { "caption": { "mouse": "空いているマスをクリックすると、選択中の原子（C）が置けます",
                      "touch": "空いているマスをタップすると、選択中の原子（C）が置けます" },
        "actions": [ { "type": "click", "x": 420, "y": 294 } ] },
      { "caption": "隣のマスに置くと、自動で結合がつながります",
        "actions": [ { "type": "click", "x": 462, "y": 294 } ] }
    ]
  }
]
```

- `actions[].type`: `click` / `drag`（from→to、伸縮や結合描画用）/ `hover`（プレビュー表示）/
  `button`（DOM要素をクリック。例: `{"type":"button","selector":"#btn-tool-bond"}`）/
  `wait`（ミリ秒）
- 座標は SVG 論理座標。再生器が getScreenCTM でクライアント座標へ変換する
  （開発方針の座標変換ルールに従う）
- 回帰テスト（tests.js）のイベント駆動ヘルパーと同じ方式なので、
  デモが動く＝その操作経路が生きていることの検証にもなる

## 4. 再生器（tutorial.js 新設）

- `TutorialPlayer`:
  - `play(tutorialId)`: 現在の作図を `serializeState()` で退避 → キャンバスを全消去 →
    ステップを順に再生 → 終了/中断時に `restoreState()` で完全復元（Undo履歴も汚さない）
  - ゴーストカーソル: ui-group の上に専用 `<div>`（SVG外のオーバーレイ。パン・ズームの
    影響を受けないようクライアント座標で動かす）。移動は 300〜500ms のイージング、
    クリック時はリング状のパルス
  - 実イベント発行は tests.js と同じ `PointerEvent`（pointerdown → pointerup、
    ドラッグは down → move×n → up）。**発行の直前にカーソルが到着**する順序を守る
  - 字幕: キャンバス下部の半透明バー。ステップごとに更新
  - 制御: ⏸一時停止 / ⏭次へ / ✕中断（Escも可）。中断はいつでも安全（最後に必ず復元）
- 再生中はユーザー入力を一時ブロック（透明オーバーレイ）。誤操作でデモが壊れないように

## 5. 呼び出し導線

1. **❓ヘルプボタン**（ヘッダー右）→ FAQ モーダル:
   - 検索ボックス（title + keywords の部分一致でフィルタ）
   - パート一覧（各行: タイトル・1行説明・「▶ デモを見る」）
   - デバイス切替セレクタ（自動判定の結果を初期値に）
2. **ボタンの長押しホバー**: 主要ボタン（ツール・モジュール・反応カード等）に
   `data-tutorial="id"` を付与。800ms ホバーで小チップ「▶ デモを見る」を表示。
   タッチ環境ではホバーが無いため、この導線は自動的に無効（FAQからのみ）
3. 邪魔にならない原則: 自動再生・初回強制ツアーはしない。すべてユーザー起点

## 6. 収録パート（初期セット）

| id | タイトル | 主な内容 |
|---|---|---|
| place-atom | 原子を置く・つなげる | クリック配置・自動結合・元素の切替 |
| bond-edit | 結合の次数変更と切断 | クリックでトグル、右クリック/長押しで切断 |
| bond-stretch | 結合を伸ばす | 結合線ドラッグ・グリッド倍数スナップ |
| ring-fusion | 環と縮合環 | モジュール配置・ゴースト・辺に吸着させる縮合 |
| functional-group | 官能基を付ける | ホバーのゴースト・置けない場合の赤表示 |
| reaction | 反応を実行する | 名称呼び出し→反応ボタン→Undoで戻す |
| view-control | 画面操作 | パン（右ドラッグ/2本指）・ズーム・全体表示 |

## 7. マイルストーン

- **M1**: 再生器＋FAQモーダル＋デバイス判定/切替＋パート3本（place-atom / bond-edit / ring-fusion）
- **M2**: 残り4パート＋ボタン長押しホバー導線＋回帰テスト（全パートを高速再生して完走・復元を検証）
- **M3**: 反応機構ビューア・クイズ系のパート追加、FAQに「よくある質問」テキスト項目の同居

## 8. リスク・未決事項

- **座標の陳腐化**: UIレイアウト変更でデモ座標がずれる → 回帰テストで全パートを
  高速再生し「期待する分子ができること」まで検証する（座標ズレを自動検出）
- **タッチ実機との差**: デモはマウス相当のイベントで再生する（タッチ固有の
  合成クリック等は説明文でカバー）。実機タッチの検証は P7-7 と同様ユーザー実機で
- **再生中の画面サイズ変更**: 再生開始時にビューをリセット（fitCanvasToTarget）して
  座標系を安定させる
